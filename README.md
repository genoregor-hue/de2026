# Демоэкзамен 2026 — Модуль 1 (автоустановка)

КОД 09.02.06-1-2026 · «Сетевое и системное администрирование».
Самодостаточный репозиторий: на каждой VM ставится своя роль одной командой,
без зависимости от внешнего сайта. Источник задания: sys.digit-shop.ru/demo/2026/m1.

## Как пользоваться

На **каждой** виртуалке:

```bash
git clone <твой-репо> de2026 && cd de2026
nano config.env          # 1. ОБЯЗАТЕЛЬНО: имена интерфейсов IF_EXT / IF_INT (см. ip -br link)
sudo ./bootstrap.sh      # 2. сам определит роль по hostname и поставит
```

Если hostname ещё дефолтный — укажи роль явно:

```bash
sudo ./bootstrap.sh hq-rtr
```

Роли: `isp` `hq-rtr` `br-rtr` `hq-srv` `br-srv` `hq-cli`.

## Что правится перед запуском

Только `config.env`. Критично — **имена сетевых интерфейсов** (`IF_EXT`, `IF_INT`,
а для ISP ещё `IF_INT2`): на разных стендах это `ens18/ens19`, `eth0/eth1` и т.п.
Узнать: `ip -br link`. Всё остальное (адреса, VLAN, пароли, порт SSH) уже выставлено
по заданию и при желании меняется там же.

## Схема адресации (по умолчанию)

| Сегмент | Сеть | Маска | Узлы |
|---|---|---|---|
| ISP→HQ | 172.16.1.0 | /28 | ISP .1, HQ-RTR .2 |
| ISP→BR | 172.16.2.0 | /28 | ISP .1, BR-RTR .2 |
| VLAN 100 (серверы, ≤32) | 192.168.100.0 | /27 | HQ-RTR .1, HQ-SRV .10 |
| VLAN 200 (клиенты, ≥16) | 192.168.200.0 | /28 | HQ-RTR .1, HQ-CLI — DHCP |
| VLAN 999 (управление, ≤8) | 192.168.99.0 | /29 | HQ-RTR .1 |
| BR-LAN (≤16) | 192.168.110.0 | /28 | BR-RTR .1, BR-SRV .10 |
| Туннель GRE | 10.10.10.0 | /30 | HQ .1, BR .2 |

## Что делает каждая роль (пункты задания)

- **isp** — uplink по DHCP, два /28 к офисам, динамический NAT (PAT). (п.2, 8)
- **hq-rtr** — net_admin; router-on-a-stick VLAN 100/200/999 на одном интерфейсе;
  NAT; GRE-туннель; OSPF только на туннеле с MD5-паролем; DHCP-сервер для VLAN 200
  (шлюз HQ-RTR, DNS HQ-SRV, суффикс au-team.irpo, адрес роутера исключён). (п.3,4,6,7,8,9)
- **br-rtr** — net_admin; LAN /28; NAT; GRE-туннель; OSPF. (п.3,6,7,8)
- **hq-srv** — sshuser (UID 2026, sudo без пароля); SSH порт 2026, только sshuser,
  MaxAuthTries 2, баннер «Authorized access only»; BIND с прямой+обратной зонами
  и форвардером 77.88.8.7. (п.3,5,10)
- **br-srv** — sshuser + SSH-хардненинг. (п.3,5)
- **hq-cli** — клиент DHCP в VLAN 200. (п.9)

Часовой пояс (п.11) ставится на всех — правь `TZ_REGION` под место проведения.

## Проверка

На сайте есть `checker` (Proxmox Guest Agent) — гоняет все 11 пунктов автоматически.
Быстрая ручная сверка:

```bash
# с HQ-CLI
ping -c1 77.88.8.8                 # интернет (NAT)
nslookup hq-srv.au-team.irpo       # DNS
ping -c1 192.168.110.10            # HQ-CLI -> BR-SRV (через туннель/OSPF)
```

## Важные оговорки

- Сеть применяется **вживую** (`ip`) для текущей сессии **и** записывается в
  **etcnet** (`/etc/net/ifaces/...`) для постоянства — после ребута адреса,
  VLAN и туннель поднимутся сами. Скрипты включают службу `network` и гасят
  NetworkManager. Сетевую службу внутри скрипта намеренно **не** перезапускаем
  (чтобы не оборвать сессию); применить etcnet немедленно: `systemctl restart network`.
- Имена сервисов/пакетов (`bind`/`named`, `dhcpd`, `frr`) могут отличаться между
  Альт Сервер / JeOS — скрипты пробуют типовые варианты.
- Модули 2 и 3 (Samba DC, RAID, NFS, Docker, LAMP, Nginx; KVM/cloud-init,
  Prometheus/Grafana) тут пока не упакованы — материалы есть на том же сайте.
