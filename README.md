# Демоэкзамен 2026 · Модуль 1 — автоустановка

КОД 09.02.06-1-2026 «Сетевое и системное администрирование».
Скрипты настройки всех 6 машин Модуля 1. Источник задания:
https://sys.digit-shop.ru/demo/2026/m1

## Структура

```
setup/        готовые скрипты под дефолтную адресацию (см. ниже)
generate.py   генератор: собирает скрипты под ТВОИ данные (адреса/интерфейсы)
docs/         гайды по модулям 1-2-3 + TROUBLESHOOTING (решение проблем)
```

## Быстрый старт (на каждой машине)

```bash
su -                             # под root, ОБЯЗАТЕЛЬНО с дефисом
git clone https://github.com/genoregor-hue/de2026.git
cd de2026
ip -br link                      # посмотреть реальные имена интерфейсов
```

Дальше два пути:

**A. Данные совпадают с дефолтом** → запускай готовый скрипт напрямую:
```bash
bash setup/isp.sh        # подставь свою роль
```

**B. Данные другие** (другие адреса/VLAN/интерфейсы — частый случай) →
собери скрипты под себя генератором:
```bash
python3 generate.py --ask        # ответит на вопросы (Enter = дефолт)
bash out/isp.sh                  # запуск пересобранного скрипта
```
В VirtualBox интерфейсы обычно `enp0s3 / enp0s8 / enp0s9`, а не `ens19/20/21` —
тогда путь B обязателен.

## Роли и порядок запуска

`isp` → `hq-rtr` → `hq-srv` → `br-rtr` → `br-srv` → `hq-cli`

Порядок важен: роутеры проверяют интернет и не настроятся, пока не поднят ISP.

## Адресация по умолчанию

| Сегмент | Сеть | Узлы |
|---|---|---|
| ISP→HQ | 172.16.1.0/28 | ISP .1, HQ-RTR .2 |
| ISP→BR | 172.16.2.0/28 | ISP .1, BR-RTR .2 |
| VLAN 100 (серверы) | 192.168.100.0/27 | HQ-RTR .1, HQ-SRV .2 |
| VLAN 200 (клиенты, DHCP) | 192.168.200.0/28 | HQ-RTR .1, HQ-CLI DHCP |
| VLAN 999 (управление) | 192.168.99.0/29 | HQ-RTR .1 |
| BR-LAN | 192.168.0.0/28 | BR-RTR .1, BR-SRV .2 |
| GRE-туннель | 10.10.10.0/30 | HQ .1, BR .2 |

Прочее: домен `au-team.irpo`, SSH порт 2026, пользователи `sshuser` (UID 2026) /
`net_admin`, пароль `P@ssw0rd`, OSPF-ключ `1245`, DNS-forwarder `77.88.8.8`.

> VLAN tag 100/200 для HQ-SRV и HQ-CLI назначается в Proxmox/VirtualBox
> на сетевой карте VM (Hardware → Network → VLAN Tag), а не внутри ОС.

## Если что-то не запускается

Смотри **docs/TROUBLESHOOTING.md** — там реальные ошибки со стенда
(sysctl, timezone, имена интерфейсов, vim, PATH) и готовые решения.

## О надёжности скриптов

Скрипты идемпотентны (можно перезапускать) и не используют `set -e` —
ошибка одной команды не валит всю настройку, в конце выводится
`=== Verification ===` со статусом каждого пункта.
