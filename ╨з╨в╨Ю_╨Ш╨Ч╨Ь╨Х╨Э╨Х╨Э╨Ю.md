## Исправления в скриптах (setup/*.sh и generate.py)

Из-за этих багов скрипты падали на стенде на середине настройки. Исправлено:

1. **Убран `set -e`** → теперь ошибка одной команды НЕ обрывает весь скрипт.
   Раньше: упал `vim`/`timedatectl`/`sysctl` — и роутер настроен наполовину.
   Теперь: скрипт доделывает всё что может и в конце показывает
   `=== Verification ===` со статусом каждого пункта.

2. **`export PATH=$PATH:/sbin:/usr/sbin`** в начале каждого скрипта.
   Чинит ошибку `sysctl: команда не найдена` (когда зашёл через su без дефиса).

3. **`vim` → `vim-console`**. Пакет `vim` на Альте виртуальный (несколько
   вариантов), его установка возвращала ошибку. `vim-console` ставится чисто.

4. **Часовой пояс через симлинк**: `ln -sf /usr/share/zoneinfo/... /etc/localtime`
   с фолбэком на timedatectl. Чинит `Failed to set time zone`.

## Итоговый рабочий процесс на экзамене

На КАЖДОЙ машине (порядок: ISP → HQ-RTR → HQ-SRV → BR-RTR → BR-SRV → HQ-CLI):

```bash
su -                              # под root, ОБЯЗАТЕЛЬНО с дефисом
git clone https://github.com/genoregor-hue/de2026.git   # один раз, потом cd
cd de2026
ip -br link                       # узнать реальные имена интерфейсов
```

Затем правишь интерфейсы в generate.py (блок CONFIG) ИЛИ через --ask, и:
```bash
python3 generate.py               # соберёт все 6 скриптов в out/
bash out/<роль>.sh                # напр. bash out/isp.sh
```

Если что-то пошло не так — docs/TROUBLESHOOTING.md.
